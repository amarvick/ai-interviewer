# Problem ID String Migration

You asked to move problem IDs to strings.  
This migration converts:
- `problems.id` from `integer` -> `varchar(64)`
- All related FK columns:
  - `submissions.problem_id`
  - `testcases.problem_id`
  - `problem_list_problems.problem_id`

## Apply migration

```bash
docker exec -it ai_interviewer_db psql -U postgres -d ai_interviewer -f /Users/alexmarvick/Desktop/Personal/Programming/ai-interviewer/backend/docs/problem_id_string_migration.sql
```

## Verify

Open psql:
```bash
docker exec -it ai_interviewer_db psql -U postgres -d ai_interviewer
```

Then run:
```sql
\d problems
\d submissions
\d testcases
\d problem_list_problems
SELECT id FROM problems LIMIT 5;
```

You should now see `varchar(64)` for all `problem_id` columns and `problems.id`.

## Notes

- Existing numeric IDs remain valid as strings (example: `1` becomes `"1"`).
- New problem IDs are generated by backend model default like `problem_ab12cd34ef56`.
- `POST /problem` now also accepts an optional custom `id` if you want to set one explicitly.
